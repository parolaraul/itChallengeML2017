El objetivo de este ejemplo es encontrar el mximo de la funcin $f(x) = x \, \sin(4x \, - \, 2$), evaluada dentro del intervalo $[-2, 3]$. Supongamos que los nmeros con los que vamos a trabajar los queremos representar con una presicin de dos dgitos decimales (es decir, no habrn valores representables entre 0 y 0.01, por ejemplo).

Para comenzar, definimos la funcin en cuestin:

In [3]: 
maximo = 3
minimo = -2
prec = 0.01

def func(x):
    return x * np.sin(4 * x - 2)

En este caso, que es un problema ficticio, podemos graficar la funcin y observar donde se encuentra el mximo.

O mejor, aplicando anlisis matemtico, podemos calcular la derivada de la funcin y calcular sus ceros, para luego ver cual de ellos es el mximo global.

$\frac{df(x)}{dx} = \frac{d \left( x\,\sin(4x\,-\,2) \right)}{dx} = 4x\,\cos(2\,-\,4x)\,-\,\sin(2\,-\,4x) = 0$

Esto permite estimar el mximo en $ x \approx 2.48852668729269 $

Una posible heurstica que podemos usar es la denominada Hill Climbing. Para usarla, debemos definir tres cuestiones fundamentales:

  1. Cmo se van a representar las potenciales soluciones al problema y de que manera se podrn generar soluciones al azar.

  2. Cmo calificar una soluciones potencial, para comparar dos soluciones representadas y poder determinar cual de las dos es mejor que la otra intentando resolver el problema (aunque ninguna de las dos realmente lo resuelva).

  3. Cmo generar a partir de una solucin dada un conjunto de nuevas soluciones vecinas a esta, lo que implica definir adems una mtrica de distancia entre soluciones.

Pero primero, implementamos el algoritmo:

In [6]: 
def hill_climb(solucion_inicial, evaluacion, obtener_vecinos):
    """
    Hill climbing determinstico.
    """
    soluciones_evaluadas = 1
    solucion_actual = solucion_inicial
    evaluacion_actual = evaluacion(solucion_actual)

    optimo_local = False
    while not optimo_local:
        vecinos = obtener_vecinos(solucion_actual)
        optimo_local = True
        for vecino in vecinos:
            evaluacion_vecino = evaluacion(vecino)
            soluciones_evaluadas += 1
            if evaluacion_vecino > evaluacion_actual:
                solucion_actual = vecino
                evaluacion_actual = evaluacion_vecino
                optimo_local = False

    return solucion_actual, soluciones_evaluadas

Si a las soluciones se las representan como valores en punto flotante, puede generarse una solucin al azar usando la funcin random() y escalando el valor al intervalo de trabajo, redondeando con round() a la cantidad de decimales que define la precisin.

def nueva():
    return round(minimo + random.random() * (maximo - minimo), 2)

Para generar soluciones dentro de la vecindad de una solucin dada, simplemente generamos 4 nuevas soluciones: $[ x-0.03, x-0.01, x+0.01, x+0.03]$. Estas cuatro soluciones tienen la particularidad que se encuentran dentro del entorno $(x \pm 0.05)$ y son valores representados con la precisin deseada.

Es importante tomar la precaucin de no generar soluciones fuera del rango permitido. Por ejemplo, si $x=3$, no deberan generarse soluciones mayores que x pues se encontraran fuera del rango fijado para el problema.

def vecinos(x):
    v = []
    d = -0.03
    for i in range(4):
        n = x + d
        if n >= minimo and n <= maximo:
            v.append(n)
        d += 0.02
    return v
